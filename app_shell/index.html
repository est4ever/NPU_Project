BATTERY_SAVER: Optimizes for efficiency by preferring NPU (Neural Processing Unit) to minimize power consumption and heat generation.

        How backends work (add and switch)
        

          

            Ships empty: downloads do not include model weights or third-party inference engines. You choose the stack; this UI only stores paths in registry/backends_registry.json. Each row needs a unique id, type, and entrypoint (relative to the project root or absolute).
          

          

            type: builtin — Use this for the included npu_wrapper build (OpenVINO GenAI). run.ps1 loads Intel OpenVINO setupvars.bat, then runs entrypoint (for example dist/npu_wrapper.exe or build/Release/npu_wrapper.exe). Skip this type if you are not using that stack.
          

          

            type: external — run.ps1 does not load OpenVINO setup; it runs entrypoint with the usual arguments (model path, --server, --port, …). Use any self-contained server you install. For this UI and npu_cli.ps1 to work unchanged, that process should expose the same HTTP API as npu_wrapper (for example /v1/chat/completions, /v1/cli/status). A bare CLI will not satisfy that unless you wrap it.
          

          

            Select Backend updates the registry and triggers an API restart: the current server exits and run.ps1 starts again with the new entrypoint. The first successful server start writes registry/npu_launch_state.json (local, not in git); without it, restart from the browser fails until you start the stack once via start_app.ps1 or run.ps1 ... --server.
          

          

            If chat or status fails to connect: nothing may be listening on port 8000 yet, or the API is still restarting after a model/backend switch. Wait a few seconds, click Refresh Status, run start_app.ps1, or check the PowerShell window running run.ps1 for a bad entrypoint, crash on load, or wrong port.

    Terminal Model Flow
    
      
    Download any Hugging Face model first, then register it in Registry Explorer.
    

      
      
        
    1. Go to huggingface.co/models and choose a model you want.
      2. 
        
    2. Enter the repository path and a local model ID below, then click Build Command.
      3. 
        
    3. Run the generated terminal command to download the model files locally.
      4. 
        
    4. Install whatever backend you plan to use (built-in wrapper + OpenVINO GenAI, llama.cpp, vLLM, your own server, etc.).
      5. 
        
    5. In Registry Explorer, import/select the model and add/select the backend.
      6. 
      
    

      
    Use a repo path such as unsloth/gemma-4-E4B-it-GGUF or another GGUF-compatible model. Your model must be downloaded locally before registry import.
    
      
    To avoid downloading all repo versions, specify a single filename (e.g., model-v1-4b-q4.gguf). Leave blank to download everything.
    
      
    Where to find the filename: Go to huggingface.co, browse to your model repo, then click the Files and versions tab. Look for a .gguf file (like model-4b-q4.gguf) and copy its exact name.
    
      
    Install the backend you want, then register it in Registry Explorer (Add Backend Wizard or manual fields under Registry Actions).
    

      
    
        
        
      
    

      
    
        
      
    

      
    
        
        
      
    
      
    .\npu_cli.ps1 -Command model -Arguments "download","meta-llama/Llama-3.1-8B-Instruct","llama-3.1-8b"
    
      
    Tip: Build Command creates a terminal download command. Copy it, run it, then import the downloaded model in Registry Explorer.
